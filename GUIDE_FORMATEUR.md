# GUIDE DU FORMATEUR / TUTEUR
## Formation en ligne sur la Scoliose avec SpineSim©

---

## 1. RÔLE DU FORMATEUR / TUTEUR

### 1.1 Missions principales
1. **Accompagner** les apprenants tout au long de leur parcours (coaching, mentorat)
2. **Évaluer** les compétences (quiz, cas cliniques, simulations SpineSim©)
3. **Animer** la communauté d'apprentissage (forum, webinaires, discussions)
4. **Adapter** le parcours aux besoins individuels (remédiation, approfondissement)
5. **Contribuer** à l'amélioration continue du contenu (retours, mises à jour)

### 1.2 Profil requis
- Chirurgien orthopédiste ou neurochirurgien **spécialiste du rachis** (≥ 5 ans d'expérience)
- Expérience en **enseignement** (DES, DIU, formation continue)
- Maîtrise des **outils numériques** de base (LMS, visioconférence, forum)
- Formation initiale à **SpineSim©** validée (certification formateur)
- Disponibilité : **4-8 heures/semaine** par cohorte de 20-30 apprenants

### 1.3 Types de rôles

| Rôle | Responsabilité | Ratio apprenants |
|------|---------------|-----------------|
| **Instructeur principal** | Conception contenu, webinaires, évaluation finale | 1 pour toute la formation |
| **Tuteur** | Suivi individuel, forum, feedback quiz | 1 pour 20-30 apprenants |
| **Expert invité** | Webinaire ponctuel, cas clinique spécialisé | Ponctuel |
| **Modérateur** | Animation forum, modération, FAQ | 1 pour 100 apprenants |

| Type | Canal | Fréquence |

**Message de bienvenue :**
> Cher(e) Dr. [Nom],
> Bienvenue dans la formation « Scoliose — Approche complète ». Je serai votre tuteur tout au long de ce parcours. N'hésitez pas à me contacter pour toute question. Je vous recommande de commencer par le Module 1 (Anatomie du rachis) et de viser 1 module par semaine. Bon parcours !

**Relance inactivité (>14 jours) :**
> Cher(e) Dr. [Nom],
> J'ai remarqué que vous n'avez pas été actif(ve) sur la plateforme depuis [X] jours. Y a-t-il des difficultés ? Votre dernier module complété est le Module [X]. Je suis disponible pour un point individuel si vous le souhaitez.

**Félicitations module complété :**
> Félicitations ! Vous avez validé le Module [X] avec un score de [X]%. [Commentaire personnalisé selon le score]. Le module suivant vous attend !

### 3.3 Gestion des difficultés

| Situation | Action recommandée |
|-----------|-------------------|
| Score quiz < 50% (2 tentatives) | Contact individuel, identifier la lacune, orienter vers des ressources |
| Inactivité > 14 jours | Relance email + notification, proposer un RDV |
| Inactivité > 30 jours | Appel téléphonique, proposer un plan de rattrapage |
| Question clinique complexe | Réponse approfondie, renvoi vers une référence bibliographique |
| Problème technique | Escalade vers le support technique |
| Contestation de note | Revue de l'évaluation, explication détaillée du barème |
| Demande de contenu supplémentaire | Orientation vers modules complémentaires ou publications |

## 4. ANIMATION PÉDAGOGIQUE

### 4.1 Webinaires

#### Format standard (1h)
| Temps | Activité |
|-------|----------|
| 0-5 min | Introduction, objectifs du webinaire |
| 5-20 min | Présentation du cas clinique (imagerie, histoire, examen) |
| 20-35 min | Discussion interactive (questions, sondage, vote) |
| 35-50 min | Démonstration SpineSim (planification chirurgicale live) |
| 50-55 min | Synthèse et messages clés |
| 55-60 min | Questions-réponses |

#### Thèmes de webinaires recommandés

| # | Thème | Module associé | Fréquence |
|---|-------|---------------|-----------|
| 1 | Lecture systématique d'une radiographie de scoliose | M7 | An 1, S3 |
| 2 | Classification de Lenke — cas pratiques | M5 | An 1, S5 |
| 3 | Indications chirurgicales vs conservatoires — algorithme | M8 | An 1, S7 |
| 4 | Planification chirurgicale 3D (avec SpineSim) | M15 + M28 | An 1, S10 |
| 5 | Complications : prévention et gestion | M18-19 | An 1, S12 |
| 6 | Scoliose de l'adulte — stratégies de prise en charge | M10 | An 1, S14 |

| 8 | SpineSim avancé — cas complexes en VR | M28 | An 2, S6 |
| 9 | Innovations et recherche | M27 | An 2, S10 |
| 10 | Préparation à la certification — simulation d'examen | M29 | Avant chaque session |

### 4.2 Animation du forum

#### Bonnes pratiques
- Répondre aux questions dans les **24-48 heures**
- Encourager les échanges entre pairs (ne pas tout résoudre soi-même)
- Poster une **question de la semaine** pour stimuler l'engagement
- **Valoriser** les contributions (badge "contributeur actif")
- **Modérer** : supprimer le spam, recadrer les hors-sujet
- **Épingler** les discussions importantes et les FAQ

#### Catégories du forum
| Catégorie | Description |
|-----------|-------------|
| 📢 Annonces | Messages officiels, mises à jour |
| ❓ Questions générales | Questions sur le parcours, la logistique |
| 🏥 Cas cliniques | Discussion de cas (anonymisés), partage d'expérience |
| 🔬 Littérature et recherche | Partage d'articles, discussion de publications |
| 🎮 SpineSim | Questions sur la simulation, partage de scores |
| 💡 Suggestions | Idées d'amélioration de la formation |

### 4.3 Évaluation et feedback

#### Quiz — Rôle du formateur
- Les quiz de module sont **corrigés automatiquement** (QCM/QCS)
- Le formateur peut consulter les **statistiques par question**
- Si taux d'échec > 40% sur une question → **revoir la question** (mal formulée ?) ou **revoir le contenu** (insuffisant ?)
- Pour les cas cliniques ouverts → **correction manuelle** avec feedback personnalisé

#### SpineSim — Évaluation
- Consulter les **rapports de simulation** de chaque apprenant
- Critères d'évaluation :
  - Précision du placement des vis (% intra-pédiculaire)
  - Qualité de la correction (angle de Cobb final)
  - Temps opératoire
  - Complications évitées
  - Utilisation appropriée des outils
- Feedback : commentaire écrit + score

#### Examen de certification
- Le formateur/instructeur principal supervise l'examen
- Vérification des conditions de proctoring
- Correction des sections de planification chirurgicale (section 4 de l'examen)
- Correction de la session SpineSim pratique (section 5)
- Validation finale du certificat

### 5.1 Fonctionnalités formateur

| Fonctionnalité | Description |
|----------------|-------------|
| **Mode spectateur** | Observer en temps réel la simulation d'un apprenant |
| **Mode co-pilote** | Prendre le contrôle à distance pour montrer un geste |
| **Création de scénarios** | Définir un cas (type de scoliose, difficulté, objectifs) |
| **Bibliothèque de scénarios** | Scénarios pré-définis par module |
| **Rapport d'évaluation** | Score automatique + zones commentables |
| **Session de groupe** | Simulation partagée (1 opérateur, N observateurs) |
| **Enregistrement/replay** | Enregistrer une session pour la rejouer comme démo |
| **Annotations 3D** | Annoter le modèle en temps réel (flèches, cercles, texte) |

## 8. CHECKLISTS

### 8.1 Checklist de démarrage de cohorte

- [ ] Accès LMS vérifié (identifiants, autorisations)
- [ ] Liste des apprenants reçue et vérifiée
- [ ] Message de bienvenue envoyé à tous les apprenants
- [ ] Webinaire d'introduction planifié (Zoom/BBB + calendrier LMS)
- [ ] Forum initialisé (message de bienvenue, catégories vérifiées)
- [ ] Scénarios SpineSim vérifiés (fonctionnels, objectifs clairs)
- [ ] Calendrier de webinaires planifié (10 sessions sur 16 semaines)
- [ ] Alertes automatiques configurées (inactivité 7j, 14j, 30j)

### 8.2 Checklist hebdomadaire

- [ ] Vérifier les alertes d'inactivité
- [ ] Répondre aux questions du forum (< 48h)
- [ ] Vérifier les résultats de quiz de la semaine
- [ ] Contacter individuellement les apprenants en difficulté
- [ ] Publier la question de la semaine sur le forum
- [ ] Préparer le webinaire si planifié

### 8.3 Checklist de fin de cohorte

- [ ] Tous les quiz évalués et feedback envoyé
- [ ] Évaluations SpineSim corrigées
- [ ] Examen de certification supervisé
- [ ] Certificats validés et envoyés

- [ ] Rapport de cohorte rédigé (statistiques, retours, suggestions)
